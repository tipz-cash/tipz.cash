## Day 1: API Setup

### Twitter API Access
- [ ] Applied at developer.twitter.com
- [ ] Project created: "TIPZ"
- [ ] App created: "tipz-verification"
- [ ] Bearer token generated
- [ ] Added to `.env.local`: `TWITTER_BEARER_TOKEN=xxx`
- [ ] Tested: registrations auto-verify (not "pending")
- [ ] Deployed to production

**Status:** ________________ (Pending / Complete)

### NEAR Testnet (Optional but Recommended)
- [ ] Testnet account created at wallet.testnet.near.org
- [ ] Account ID: ________________
- [ ] Private key exported
- [ ] Added to `.env.local`: `NEAR_ACCOUNT_ID=xxx`
- [ ] Added to `.env.local`: `NEAR_PRIVATE_KEY=xxx`
- [ ] Set: `NEAR_DEMO_MODE=false`
- [ ] Funded with testnet NEAR (~10 NEAR from faucet)
- [ ] Tested tip flow end-to-end

**Status:** ________________ (Skipped / Complete)

## Day 2-3: Creator Outreach

### Preparation
- [ ] Reviewed outreach templates in `/docs/outreach/seed-creator-list.md`
- [ ] Identified 15-20 potential creators to contact
- [ ] Filled in target list with real handles

### Outreach
- [ ] Day 2 AM: Sent first batch of 5-7 DMs
- [ ] Day 2 PM: Sent second batch of 5-7 DMs
- [ ] Day 3: Followed up with non-responders
- [ ] Day 3: Sent third batch of 5-7 DMs

### Results
- [ ] **At least 5 creators registered** (minimum for grant)
- [ ] **Ideally 10+ registered** (strengthens application)
- [ ] Updated seed-creator-list.md with statuses
- [ ] Collected any feedback/testimonials

**Registered count:** _____ / 10 target

---

## Day 3: Demo Video

### Recording
- [ ] Screen recording software ready (OBS/QuickTime/Loom)
- [ ] Script reviewed in `/docs/grants/PRE-GRANT-SETUP.md`
- [ ] Tested all flows work before recording
- [ ] Recorded video (aim for < 3 minutes)
- [ ] Uploaded to YouTube (unlisted) or Loom

**Video URL:** ________________

### Video Checklist
- [ ] Shows homepage / value prop
- [ ] Shows creator registration flow
- [ ] Shows tip page experience
- [ ] Shows browser extension
- [ ] Explains privacy benefit
- [ ] Clear audio, no awkward pauses

---

## Day 4: Grant Application

### Preparation
- [ ] Reviewed `/docs/grants/zcash-grant-application.md`
- [ ] Filled in [X] placeholders with real data
- [ ] Updated creator count
- [ ] Added demo video link
- [ ] Added GitHub link (if public)
- [ ] Reviewed budget breakdown

### Personal Info
- [ ] Filled in team section with your background
- [ ] Added contact information
- [ ] Added X/Twitter handle
- [ ] Added email address

### Review
- [ ] Read application aloud (catches errors)
- [ ] Checked all links work
- [ ] Verified milestone dates are realistic
- [ ] Had someone else proofread (optional but helpful)

## Post-Submission

### Monitoring
- [ ] Set calendar reminder to check forum daily (first week)
- [ ] Set calendar reminder for ZCG response (typically 2-4 weeks)
- [ ] Continue creator outreach while waiting

### If Asked for Updates
- [ ] Prepared to provide additional info quickly
- [ ] Have traction metrics ready to share
- [ ] Can do live demo if requested
